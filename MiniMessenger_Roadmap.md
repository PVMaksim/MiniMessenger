# 💬 MiniMessenger  
## Техническое задание и дорожная карта разработки  
**Версия 1.0 • 2025**

> **Цель проекта:** создать простой, надёжный мессенджер для небольшой группы пользователей (до 50 человек) с текстовыми сообщениями и обменом файлами. Приложение должно работать в браузере, на iOS и Android. Стек: JavaScript везде (Node.js + React + React Native).

---

## 1. Обзор проекта

### 1.1 Целевая аудитория
Приложение рассчитано на закрытую группу пользователей — например, команда, семья, друзья. Максимум ~50 активных пользователей. Доступ только по приглашению, без публичной регистрации.

### 1.2 Технологический стек

| Уровень | Технология | Назначение |
|---------|-----------|-----------|
| Бэкенд (сервер) | Node.js + Express | API, авторизация, логика чатов |
| Real-time связь | Socket.io | Мгновенная доставка сообщений |
| База данных | PostgreSQL | Хранение пользователей и сообщений |
| Кэш / сессии | Redis | Быстрые данные, онлайн-статус |
| Файлы | Multer + локальное хранилище | Загрузка фото и документов |
| Веб-клиент | React | Браузерная версия приложения |
| Мобильный клиент | React Native | iOS и Android из одного кода |
| Деплой | Docker + Nginx + VPS | Запуск на сервере |

---

## 2. Функциональные требования

### MVP — Минимально жизнеспособный продукт
*Это то, без чего приложение не имеет смысла. Всё перечисленное ниже должно быть реализовано в первую очередь.*

1. Регистрация и вход по логину + паролю
2. Список контактов / пользователей системы
3. Личные чаты (1 на 1) с историей сообщений
4. Отправка и получение текстовых сообщений в реальном времени
5. Отправка изображений и файлов (до 20 МБ)
6. Индикатор «онлайн / офлайн» у пользователей
7. Уведомление о новых сообщениях (в браузере и приложении)

### Версия 2 — После MVP
*Эти функции можно добавить после того, как базовый мессенджер будет работать.*

1. Групповые чаты (несколько участников)
2. Реакции на сообщения (эмодзи)
3. Ответ на конкретное сообщение (reply)
4. Поиск по сообщениям
5. Аватары пользователей
6. Просмотр медиафайлов галереей

## 3. Дорожная карта разработки

> Весь проект разбит на 5 фаз. Каждая фаза завершается рабочим результатом, который можно потрогать и проверить.

### ▶️ Фаза 1: Подготовка и настройка окружения ⏱ 1–2 дня

**Что делаем:**
- [ ] Устанавливаем Node.js, PostgreSQL, Redis на локальную машину
- [ ] Настраиваем редактор кода (VS Code с нужными расширениями)
- [ ] Создаём структуру папок проекта
- [ ] Инициализируем git-репозиторий для хранения кода

> 🎯 **Результат фазы:** на компьютере установлено всё необходимое, создан пустой проект с правильной структурой папок.

---

### ▶️ Фаза 2: Бэкенд — сервер и база данных ⏱ 1–2 недели

**Что делаем:**
- [ ] Проектируем схему базы данных (таблицы: `users`, `chats`, `messages`, `files`)
- [ ] Создаём REST API для регистрации, входа, получения истории
- [ ] Настраиваем JWT-авторизацию (безопасные токены доступа)
- [ ] Подключаем Socket.io для real-time обмена сообщениями
- [ ] Реализуем загрузку файлов на сервер
- [ ] Пишем базовые тесты API через Postman

> 🎯 **Результат фазы:** работающий сервер, к которому можно подключиться и отправить сообщение через инструмент Postman.

---

### ▶️ Фаза 3: Веб-клиент (браузер) ⏱ 1–2 недели

**Что делаем:**
- [ ] Создаём React-приложение с нуля (или используем Vite)
- [ ] Делаем экран входа и регистрации
- [ ] Делаем главный экран: список чатов слева, переписка справа
- [ ] Подключаем Socket.io на фронтенде — сообщения приходят мгновенно
- [ ] Реализуем отправку и просмотр файлов и изображений
- [ ] Добавляем индикатор онлайн-статуса
- [ ] Тестируем в двух браузерах одновременно

> 🎯 **Результат фазы:** можно открыть два окна браузера и переписываться между ними в реальном времени с файлами.

---

### ▶️ Фаза 4: Мобильное приложение (iOS + Android) ⏱ 2–3 недели

**Что делаем:**
- [ ] Инициализируем проект React Native через Expo (самый простой способ)
- [ ] Переиспользуем логику из веб-версии (авторизация, API-запросы)
- [ ] Делаем мобильный интерфейс: список чатов и экран переписки
- [ ] Подключаем push-уведомления через Expo Notifications
- [ ] Реализуем выбор файлов из галереи и камеры
- [ ] Тестируем на симуляторе iOS и Android

> 🎯 **Результат фазы:** рабочее мобильное приложение, установленное на телефон через Expo Go или собранное как `.ipa` / `.apk` файл.

---

### ▶️ Фаза 5: Деплой и запуск на сервере ⏱ 2–3 дня

**Что делаем:**
- [ ] Арендуем VPS (например Hetzner или DigitalOcean, ~5–10 USD/мес)
- [ ] Упаковываем приложение в Docker-контейнеры
- [ ] Настраиваем Nginx как обратный прокси
- [ ] Подключаем SSL-сертификат (Let's Encrypt, бесплатно)
- [ ] Настраиваем автоматическое резервное копирование базы данных
- [ ] Проверяем работу с реальными устройствами

> 🎯 **Результат фазы:** приложение доступно по постоянной ссылке в интернете, работает 24/7, данные не теряются.

## 5. Структура папок проекта

```
minimessenger/
├── backend/                 # Серверная часть (Node.js)
│   ├── src/routes/          # API-роуты (auth, chats, messages)
│   ├── src/models/          # Модели базы данных
│   ├── src/socket/          # Логика Socket.io
│   ├── uploads/             # Загруженные файлы пользователей
│   └── server.js            # Точка входа сервера
│
├── frontend/                # Веб-приложение (React)
│   ├── src/components/      # Переиспользуемые компоненты
│   ├── src/pages/           # Страницы (Login, Chat)
│   ├── src/api/             # Запросы к серверу
│   └── src/socket/          # Клиентский Socket.io
│
├── mobile/                  # Мобильное приложение (React Native)
│   ├── screens/             # Экраны приложения
│   ├── components/          # Компоненты интерфейса
│   └── app.json             # Конфигурация Expo
│
├── docker-compose.yml       # Запуск всех сервисов
└── README.md                # Документация проекта
```
