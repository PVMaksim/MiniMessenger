## Архитектурные решения и почему

| Решение | Почему |
|---------|--------|
| **Rate limiting без express-rate-limit** | In-memory Map, нет лишних зависимостей. Для 50 пользователей достаточно. |
| **Моки в тестах через `__mocks__/`** | Jest автоматически подхватывает папку `__mocks__` рядом с `src`. |
| **Двухстадийный Dockerfile** | DevDependencies не попадают в продакшен-образ, размер меньше. |
| **nginx rewrite `/api/` → `/`** | Бэкенд не знает про `/api` — его роуты остаются чистыми. |
| **Socket.io через nginx** | Заголовки `Upgrade` и `Connection: upgrade` обязательны для WebSocket. |
| **Уведомления без внешних библиотек** | `https` модуль встроен в Node.js — меньше зависимостей. |
| **Бэкап через docker exec** | Не требует установки pg_dump на хост-машине. |

### Обязательные

| Переменная | Пример | Примечание |
|-----------|--------|-----------|
| `JWT_SECRET` | 64 hex-символа | Минимум 32 символа, не дефолт |
| `DB_HOST` | `postgres` (Docker) / `localhost` | |
| `DB_NAME` | `minimessenger` | |

## Следующая сессия — план

### Приоритет: Высокий (для продакшена)

1. **Закоммитить изменения в Git**
   ```bash
   git add .
   git commit -m "feat: добавить уведомления об ошибках и систему бэкапов"
   ```

2. **Установить зависимости для мобайла**
   ```bash
   cd mobile
   npx expo install expo-image-picker expo-document-picker
   ```

3. **Настроить Telegram-уведомления**
   - Создать бота через @BotFather
   - Получить ADMIN_TELEGRAM_ID через @userinfobot
   - Заполнить `backend/.env`

4. **Настроить cron для бэкапов**
   ```bash
   crontab -e
   0 3 * * * /opt/minimessenger/scripts/backup.sh >> /var/log/minimessenger_backup.log 2>&1
   ```
